作成したアカウントの申告

このサイトを複数人で編集する場合、あなたのアカウントに対してGitHubのvr-yamagata/vr-yamagata.github.ioリポジトリにpushやmergeができる権限(collaborator権限)を付与する必要があります。

ちなみに、編集者はpull requestを管理者に送信して、管理者が内容をチェックしてmergeする方法もありますが、Git初心者には敷居が高いのと趣味プログラムの範囲で使用する機会は少ないという理由で採用しません。

代表のTwitterなりLINEなりDiscordなりで、「VR部のリポジトリのコラボレーターに追加してください！アカウント名は〇〇です！」と連絡してください。追加します。

編集方法

自分のPCにリポジトリをcloneする

このサイトの中身を自分のPCにコピーします。

ただ、コピーされるのはGit管理されたファイルで、サイトの変更履歴やブランチの情報なども含まれるので、コピーとかダウンロードといわずにclone（クローン）といいます。

cloneすることによって、ローカルで編集した後のファイルをGitHubに反映させる(pushという)時も、特別な設定をすることなくコマンド一発でできるようになります。

cloneするには、コマンドプロンプトまたはターミナルに

git clone git@github.com:vr-yamagata/vr-yamagata.github.io.git

と打ち込んで(コピペ推奨)、エンターキーを押します。

GitとGitHubの連携の設定がうまくできていれば、ルートディレクトリにvr-yamagata.github.ioというフォルダができているはずです。

なお、カレントディレクトリを変更してからcloneすれば任意の場所にフォルダを作成することができます。

詳細を知りたい人はProgateのコマンドライン編を履修しましょう。

ファイルを編集する

編集したいファイルをエディタで開いてください。/Users/UserName/vr-yamagata.github.io/ 的な場所にあるはずです。

index.htmlをダブルクリックすると、既定のブラウザで開くと思います。

テキストエディタで変更した内容がすぐに確認できるので、このブラウザの画面を見ながら開発を進めましょう。

いちばん上の階層にあるindex.htmlを右クリックして（index.htmlは複数ある）、メモ帳なりSublimeTextなりで開いて、適当に編集します。

まずは自分の名前をメンバー一覧に追記してみましょう。

編集したら[ctrl] + Sキーでファイルを上書き保存して、ブラウザを[ctrl] + Rキーで更新してください。

これらの動作は頻繁にするので、ショートカットキーを使いましょう。

変更が反映されたのを確認したら、コマンドライン上で

git add .
git commit -m "メンバー一覧に〇〇を追記"

と打ちます。一行ごとにエンターキーを押してください。

addコマンドで変更したファイルをGitに認識させて、commitコマンドで記録しています。

ダブルクォートの中身をコミットメッセージといいます。

どんな変更をしたのかを簡潔に記載してください。

最後に、

git push

とコマンドラインに打ち込むと、これまで記録された内容がGitHubサーバー上にアップロードされ、Webサイトの内容が書き換わります。

これで変更の一連の手順は終わりです。

Progateで学んだことを活かして、個人ページを充実させていきましょう。

補足・個人ページの作り方

個人ページを作るには、新しいファイルを作成する必要があります。

① membersフォルダの中に個人名でフォルダを作成する

② 作成したフォルダの中に、index.htmlを作成する

③ VR部のホームページ(一番上の階層のページ)に、そのページへのリンクを追記する

例: <a href ="./members/naoto">髙木の個人ページ</a>

ちなみに、本来、./members/naotoは./members/naoto/index.htmlです。

たいていのWebサーバーはファイル名が省略されていた場合index.htmlのことなんだなと忖度してくれるので大丈夫という仕組みです。

そのため、ローカルでは挙動が違うはずですが、問題ありません。

なお、この要領でページを増やしていくことが可能です。

ドメインが変わっても良いように相対パスで指定してください。
